# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Stellopay Core, please report it privately by emailing **security@stellopay.com**.

Do **not** open a public GitHub issue for security vulnerabilities.

We will acknowledge receipt within 48 hours and provide an estimated timeline for a fix. Once the fix has been shipped, we will publish a security advisory.

## Scope

This security policy applies to all code in this repository, including on-chain smart contracts, off-chain tooling, scripts, and documentation examples.

## Best Practices

- Never commit private keys, RPC endpoints, wallet seeds, or production secrets.
- Review all contract upgrades and migrations carefully before deploying to mainnet.
- Use test networks for development and integration testing.
